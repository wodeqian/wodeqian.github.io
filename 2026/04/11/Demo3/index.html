SvelteKit 框架

Svelte框架

整体目录

Project/
├── src/ # 源代码
├── drizzle/ # 数据库迁移脚本 (12个版本)
├── scripts/ # 工具脚本 (种子数据、补丁等)
├── tests/ # 测试文件
├── docs/ # 文档
├── mcp/ # MCP配置
├── .svelte-kit/ # SvelteKit生成文件
├── build/ # 构建输出(Bun生产环境)

核心源码结构 (src/)

1️⃣ routes/ - 路由层 (SvelteKit路由)

(app)/ - 认证后的主应用

(app)/
├── dashboard/ # 仪表板(数据可视化)
├── orders/ # 订单管理
├── products/ # 产品资料(PDM)
├── inventory/ # 库存管理
├── settings/ # 系统设置(OAuth等)
├── +layout.server.ts # 布局服务器逻辑(身份验证)
└── +layout.svelte # 布局组件

api/ - API端点

api/
├── auth/ # 认证API
├── dashboard/ # 仪表板数据API(9个子模块)

├── uploads/ # 文件上传API(OSS)
├── oauth/ # OAuth授权API
├── integrations/ # 第三方集成

auth/ - 认证页面

auth/
├── login/ # 登录页
├── register/ # 注册页
└── …

前端部分

lib/

后端部分 (lib/server/) - DDD分层架构

server/
├── application/ # 📦 应用层(用例编排)
│ ├── auth/ # 认证服务
│ ├── order/ # 订单服务
│ ├── product-pdm/ # 产品服务
│ ├── dashboard/ # 仪表板服务(13个模块)
│ ├── attachments/ # 附件服务
│ ├── dingtalk/ # 钉钉集成
│ └── oauth/ # OAuth服务

│ ├── order/ # 订单领域
│ ├── product/ # 产品领域
│ ├── inventory/ # 库存领域
│ ├── user/ # 用户领域
│ ├── oauth/ # OAuth领域
│ └── shared/ # 共享领域逻辑

├── infrastructure/ # 🔧 基础设施层
│ ├── database/ # PostgreSQL + Drizzle ORM
│ ├── cache/ # Redis缓存
│ ├── oss/ # 阿里云OSS存储
│ ├── dingtalk/ # 钉钉API集成

├── shared/ # 📎 共享模块

└── config/ # ⚙️ 配置

🔑 关键文件说明
文件 作用
hooks.server.ts 全局服务端钩子(请求拦截、身份验证)
svelte.config.js SvelteKit配置(使用Bun适配器)
drizzle.config.ts Drizzle ORM数据库配置
vite.config.ts Vite构建配置
.env.development 开发环境变量(PostgreSQL、Redis、OSS等)
🏛️ 架构特点
DDD分层架构

┌─────────────────────────────┐
│ Routes (表示层) │ ← SvelteKit页面/API路由
├─────────────────────────────┤
│ Application (应用层) │ ← 业务用例编排
├─────────────────────────────┤
│ Domain (领域层) │ ← 核心业务逻辑
├─────────────────────────────┤
│ Infrastructure (基础设施层) │ ← 数据库/缓存/OSS/钉钉
└─────────────────────────────┘

技术栈总结
前端: Svelte 5 + SvelteKit + TailwindCSS
后端: Bun + SvelteKit SSR
数据库: PostgreSQL + Drizzle ORM
缓存: Redis
存储: 阿里云OSS
集成: 钉钉开放平台
部署: svelte-adapter-bun (单体应用)
这是一个全栈一体化的企业级OMS系统，采用清晰的DDD分层设计，便于维护和扩展。
